# Personalize in ~20 minutes

Re-skinning this storefront is a **JSON edit, not a code change**. All branding and the
product catalog live in `/public/config/*.json`, validated by Zod — a malformed re-skin
fails loudly with a clear message instead of rendering broken.

Two example skins ship in [`public/config/`](../public/config): a coffee shop
(`brand.coffee.json`, emoji logo) and a sneaker store (`brand.sneakers.json`, image logo).
`brand.json` is the **active** brand the app loads.

---

## The ~20-minute checklist

1. **Point the app at your brand (≈2 min).** Edit `public/config/brand.json` — or copy one
   of the examples as a starting point:
   ```bash
   cp public/config/brand.sneakers.json public/config/brand.json
   ```
2. **Set the identity (≈3 min).** In `brand.json`: `name`, `welcomeLine`, a logo
   (`logoEmoji` for zero assets, or `logoUrl` for a real image), and `accentColor`. The
   accent drives the primary button; button text auto-contrasts (light/dark) for legibility.
3. **Currency (≈1 min).** `currency.label` (e.g. `"TON"`) and an optional static
   `currency.usdRate` for the "≈ $X" price hint.
4. **Point at a products file (≈1 min).** `productsFile` → an absolute path under
   `/public`, e.g. `"/config/products.sneakers.json"`.
5. **Edit the catalog (≈8 min).** In that products file, list your products (see fields
   below). Prices are in TON.
6. **Images (≈3 min).** Either keep the generated SVG illustrations, or drop in **real
   photos** — see _Swapping in real photos_ below (zero code).
7. **Verify (≈1 min).** `npm run dev` and reload. If a field is wrong, the console/validation
   tells you exactly which one.

That's the whole re-skin — no component, hook, or build change.

## `brand.json` fields

| Field | Required | Notes |
| --- | --- | --- |
| `name` | ✅ | Shop name — header + document title. |
| `welcomeLine` | ✅ | One-line greeting on the catalog. |
| `logoUrl` | ▢¹ | Local image path (e.g. `/brand/logo.svg`). Bundled, **not hotlinked**. |
| `logoEmoji` | ▢¹ | Emoji fallback (e.g. `"☕"`) when there's no image. |
| `accentColor` | ✅ | Hex (`#9a5b34` or `#abc`). Primary button + highlights. |
| `currency.label` | — | Defaults to `"TON"`. |
| `currency.usdRate` | — | Positive number; static TON→USD for the price hint. Omit to hide it. |
| `productsFile` | ✅ | Absolute path starting with `/`, e.g. `/config/products.coffee.json`. |

¹ At least one of `logoUrl` / `logoEmoji` is required.

## Product fields (`products` array, ≥ 1 item)

| Field | Required | Notes |
| --- | --- | --- |
| `id` | ✅ | Stable, unique — also the cart key and image-filename convention. |
| `name` | ✅ | |
| `description` | ✅ | |
| `priceTon` | ✅ | Positive number, price in TON. |
| `image` | ✅ | **Local** absolute path (starts with `/`). Remote `http(s)://` URLs are rejected. |
| `category` | ✅ | Drives the catalog filter chips. |
| `badge` | — | Optional merchandising tag, e.g. `"Best seller"`. |

## Swapping in real photos (zero code)

1. Drop your image files anywhere under `/public`, e.g. `public/img/products/air-runner.jpg`
   → it's served at `/img/products/air-runner.jpg`.
2. Point each product's `image` at that path:
   ```json
   { "id": "air-runner", "name": "Air Runner", "image": "/img/products/air-runner.jpg", … }
   ```
3. Reload. No component or build change.

The only rule (enforced by validation): `image` must be a **local absolute path** — the demo
keeps zero external image dependencies, so hotlinked URLs are rejected. Use square images
(they render in a 1:1 frame).

The shipped SVGs are generated by `npm run gen:images` (see `scripts/`); you can ignore that
script entirely once you're using your own photos.

## Deploying under your own domain (2 extra minutes)

Two files are **per-deploy** (not per-brand) and carry the demo's origin — update them when
you host your re-skin somewhere else:

1. **`public/tonconnect-manifest.json`** — `url` → your deployed HTTPS origin, `name` → what
   the wallet shows on its connect/approve sheet, `iconUrl` → an icon on your origin.
   **The icon must be a PNG** (TON Connect does not accept SVG); `public/icon-180.png`
   ships as a ready 180×180 template.
2. **`index.html`** — the `og:url` / `og:image` link-preview tags pin the same origin.

Then set the deploy-time env per [`.env.example`](../.env.example):
`VITE_TON_RECIPIENT_TESTNET`, `VITE_BOT_URL`, and (optional) `VITE_TONCENTER_API_KEY`.
